# Challenges

## 1. Prioritization without data

You have no analytics, no A/B testing, no user interviews recorded. Every decision about what to build next is based on gut feeling and Alex's feature requests. You know this isn't sustainable, but setting up proper analytics feels like a yak-shave when there are features to ship.

## 2. Doing everything alone

You are the developer, designer, marketer, support agent, and PM. Context switching between "fix this CSS bug" and "write copy for the landing page" and "reply to Alex's email" eats most of your productive hours. You've thought about using Claude Code to parallelize some of this, but haven't figured out how to delegate effectively to an AI agent.

## 3. Burnout risk

You work on this evenings and weekends alongside a day job. The initial excitement is fading. The product works, but nobody is paying, and the gap between "working MVP" and "business" feels enormous. You've started skipping days.

## 4. Technical debt

The codebase was written fast. No tests, inconsistent naming, some copy-pasted code. Refactoring feels unproductive ("who am I refactoring for — my 3 users?"), but every new feature is harder to add than the last one.

## 5. The monetization question

You don't know what to charge, who to charge, or when to start charging. Free feels safe. But free also means no validation that this is worth anything.
